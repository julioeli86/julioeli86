Developing Cybersecurity Recommendations

For business leaders, quantifying cyber risk allows risk managers in other parts of the enterprise to roll that risk up into the enterprise-wide risk tolerance.

This report focuses on how cybersecurity teams can analyze user behavior to detect potentially suspicious patterns, that can become a Cybersecurity Risk. Cybersecurity risk is defined as insecurity of information or technology and relates to the loss of data control, integrity, and confidentiality of information systems. As well as considers the adverse implications of such actions to organizations, companies, and government operations (Editor, 2022).

The term vulnerability is defined as the weakness in a system’s security network that causes unauthorized access when triggered by a threat source (CSRC, 2022) (Reciprocity, 2022). A summary of the main cybersecurity threats and corresponding definitions is presented in Table 1, (European, 2022).

Table 1 summarizes threats by category without fully developing each of them. For example, the Malware category can be subdivided into Viruses, Worms, Trojan, Spyware, Adware, Ransomware, and Fileless malware (Cisco, 2022).

Organizations looking to detect potentially suspicious patterns can use User and Entity Behavior Analytics (UEBA) to detect users’ suspicious activities and anomalies (Palo, 2022). Table 2 presents a list of where user data can be collected within an organization’s IT network (Lord, 2018).

Microsoft provides a service called Microsoft Defender for Cloud Apps as a part of its Microsoft 365 Defender. This service helps to detect user suspicious behavior by providing a user-friendly guide to configure Built-In anomaly detection policies that are preconfigured for common security use cases (Curwin, 2022).

The cybersecurity risk can be calculated using the following formula, (Tunggal, 2022):

Cyber Risk = Threat X Vulnerability X InformationValue

The definition and concept of Return on Security Investment (ROSI) influence the investment made to ensure a company or organization is safe against cybersecurity threats. By investing in cybersecurity, a company won’t produce a profit. The main benefit will be a reduction or mitigation of cybersecurity risks. A cybersecurity investment will provide a loss reduction (ENISA, 2012).

Single Loss Expectancy (SLE) = Amount of money expected to be lost when a threat event occurs.

Annual Rate of Occurrence (ARO) = Probability that a threat event occurs in one year.

Annual Loss Expectancy (ALE) = Annual monetary loss from a threat event in a specific asset.

ALE = ARO * SLE
